# Blink Guardian - Product Spec

## 产品定位

为干眼症患者（尤其是不自觉减少眨眼频率的人群）开发的一款**极低资源占用**的浏览器端眨眼频率监测应用。通过摄像头实时检测眨眼行为，在眨眼频率过低时主动提醒，帮助用户养成健康的用眼习惯。

## 目标用户

- 长时间面对电脑的办公人群
- 已确诊干眼症患者
- 戴隐形眼镜的用户
- 游戏玩家、程序员等"忘眨眼"高风险群体

## 核心技术方案

### 眨眼检测：Eye Aspect Ratio (EAR) + MediaPipe Face Landmark

使用 EAR 算法判断眨眼，公式：
```
EAR = (|p2-p6| + |p3-p5|) / (2 * |p1-p4|)
```
- EAR < 阈值（约 0.2-0.25）→ 判定为闭眼
- EAR 恢复 → 判定为一次完整眨眼

### 资源优化策略（核心要求）

| 优化项 | 方案 |
|--------|------|
| 摄像头分辨率 | 320x240（默认），最高 640x480 |
| 检测帧率 | 3-5 FPS（非全帧率处理） |
| 模型选择 | MediaPipe Face Landmark Lite（468 点，~2ms/帧） |
| 渲染策略 | CSS 动画替代 JS 动画，数据更新节流 1s |
| 后台处理 | 页面不可见时自动暂停检测 |
| 数据存储 | localStorage，无网络请求 |
| UI 更新 | 1秒间隔更新显示数值，避免高频 DOM 操作 |

## 功能规划

### P0 - 核心功能（必须实现）

**1. 摄像头接入**
- 请求摄像头权限，低分辨率采集
- 摄像头状态指示（开启/关闭/无权限）
- 隐藏的视频元素（不显示画面，仅用于检测）
- 用户可选择显示/隐藏摄像头预览小窗

**2. 眨眼检测引擎**
- MediaPipe Face Landmark 加载（CDN，懒加载）
- 实时提取眼部关键点
- EAR 计算 + 眨眼事件判定
- 检测帧率动态调节（3-5 FPS）

**3. 实时监测面板**
- 当前眨眼频率（次/分钟）
- 眨眼状态指示（正常/偏低/警告）
- 当次会话时长
- 累计眨眼次数
- 状态颜色编码（绿色=正常，黄色=偏低，红色=警告）

**4. 提醒系统**
- 眨眼频率低于阈值时视觉提醒（屏幕边缘发光/脉冲）
- 可配置提醒间隔（避免频繁打扰）
- 声音提醒开关（可选）
- 浏览器 Notification API（页面不可见时）

**5. 会话数据记录**
- 每次使用自动记录为一个 Session
- 记录内容：开始时间、结束时间、平均眨眼率、眨眼总次数
- 存储在 localStorage
- 最近 30 天历史可查

### P1 - 增强功能

**6. 数据分析面板**
- 日/周/月平均眨眼率趋势图
- 每日使用时长统计
- 高风险时段识别
- 进步追踪（周对比）

**7. 20-20-20 规则定时器**
- 每 20 分钟提醒看 20 英尺外 20 秒
- 独立于眨眼提醒
- 可暂停/跳过

**8. 设置与个性化**
- 眨眼频率阈值配置（默认 15 次/分钟）
- 提醒方式选择（视觉/声音/通知）
- 检测灵敏度调节
- 主题切换（亮色/暗色）

### P2 - 锦上添花

**9. 数据导出** - JSON/CSV 格式导出历史数据
**10. PWA 支持** - 可安装到桌面，离线可用

## 用户故事

1. 作为一名程序员，我希望在长时间编码时被提醒眨眼，以预防干眼症加重
2. 作为干眼症患者，我希望看到我的眨眼频率趋势，了解用眼习惯是否在改善
3. 作为注重隐私的用户，我希望所有数据处理都在本地完成，不上传任何信息
4. 作为低配电脑用户，我希望这个应用几乎不占用系统资源，不影响其他工作

## 视觉风格

- **关键词**：医疗健康、安静、不干扰、专业感
- **色调**：柔和的蓝绿色系（护眼舒适），深色模式为暗灰+青色
- **布局**：最小化 UI，类似桌面小组件/状态栏，不遮挡工作区域
- **交互**：静默运行，仅在需要提醒时引起注意
- **参考**：类似 macOS 菜单栏工具（随时可见但不碍事）

## 技术选型

| 类别 | 选择 | 理由 |
|------|------|------|
| 框架 | React 18 + Vite + TS | 快速开发，类型安全 |
| 眨眼检测 | MediaPipe Face Landmark | 浏览器端最佳方案，精度高 |
| 样式 | CSS Modules + CSS Variables | 零依赖，极致轻量 |
| 图表 | 轻量 Canvas 手绘 | 避免引入 Chart.js 等重库 |
| 状态管理 | React useState/useReducer | 足够简单，无需 Redux |
| 存储 | localStorage | 无需后端，纯本地 |
| 构建体积目标 | < 200KB (gzipped) | 不含 MediaPipe 模型 |

## 约束

- **隐私优先**：所有数据本地存储，摄像头数据不离开设备
- **零后端**：纯前端应用，可部署为静态页面
- **资源极简**：CPU 占用 < 5%，内存 < 100MB
- **浏览器兼容**：Chrome/Edge 90+（需要 getUserMedia + WebAssembly）
